# 개발자 마케팅 플레이북 — DevRel, 문서화 & 커뮤니티 퍼널

엔지니어는 기존 마케팅에 반응하지 않습니다. 실제로 효과 있는 방법은 이것입니다.

## DevRel 프로그램 설계

채택을 촉진하는 역할, 지표, 활동:

- **핵심 역할**: 개발자 애드보킷 (콘텐츠+커뮤니티), DX 엔지니어 (DX+SDK), 커뮤니티 매니저 (인게이지먼트+지원)
- **핵심 지표**: 첫 Hello World까지 시간, 문서 페이지뷰, 개발자당 API 호출 수, 커뮤니티 성장률
- **활동 매트릭스**: 컨퍼런스 발표 (인지도), 튜토리얼 (활성화), 오피스 아워 (리텐션), 해커톤 (확장)
- **채용 프로파일**: 코딩도 되고 커뮤니케이션도 되는 전직 개발자; 순수 마케터는 피하세요
- **예산 배분**: 40% 이벤트/출장, 30% 콘텐츠 제작, 20% 도구/인프라, 10% 굿즈

## 문서화를 성장 엔진으로

훌륭한 문서가 최고의 사용자 획득 채널이 되는 원리:

- **SEO 금광**: 각 API 엔드포인트 = "[액션] + [언어]" 쿼리의 잠재적 랜딩 페이지
- **인터랙티브 예제**: 문서 내 실행 가능한 코드 (CodeSandbox, Replit 임베드) → 인게이지먼트 3배
- **점진적 공개**: 퀵 스타트 (5분) → 튜토리얼 (30분) → 레퍼런스 (종합)
- **기여 모델**: 커뮤니티가 GitHub PR로 문서 개선 → 오너십 → 애드보커시
- **지표**: 문서→가입 전환율, 체류 시간, 검색→성공률

## API 개발자 경험

첫 API 호출까지의 시간을 줄이는 DX 개선:

- **제로 설정 온보딩**: 자동 생성 API 키, 미리 채워진 예제, 복사-붙여넣기 가능
- **SDK 품질**: 첫날부터 상위 5개 언어 지원; OpenAPI 스펙에서 자동 생성
- **에러 메시지**: 수정 제안과 관련 문서 링크가 포함된 사람이 읽을 수 있는 에러
- **플레이그라운드/샌드박스**: 커밋 전에 테스트할 수 있는 인터랙티브 API 익스플로러
- **체인지로그 규율**: 시맨틱 버저닝, 마이그레이션 가이드, 브레이킹 체인지 90일 전 예고

## 커뮤니티 퍼널

Discord/Slack → 컨트리뷰터 → 챔피언 → 고객 파이프라인:

- **스테이지 1 — 잠복자**: 서버 가입, 메시지 읽기, 콘텐츠 소비
- **스테이지 2 — 참여자**: 질문하기, 다른 사람에게 답변, 프로젝트 공유
- **스테이지 3 — 컨트리뷰터**: PR 제출, 튜토리얼 작성, 양질의 버그 리포트
- **스테이지 4 — 챔피언**: 외부에서 옹호, 발표, 뉴커머 멘토링
- **스테이지 5 — 고객**: 팀/회사가 도입할 때 유료 전환; 내부 바이인을 챔피언

## 해커톤 ROI

해커톤이 언제 효과적이고, 어떻게 운영하며, 임팩트를 어떻게 측정하는지:

- **스폰서해야 할 때**: 프로덕트가 개발자가 그 위에 빌드하는 플랫폼일 때만
- **포맷**: 스타터 템플릿 제공, 전용 멘토 트랙, 최우수 통합상
- **측정**: API 사용 프로젝트 수, 해커톤 후 리텐션 (30일간 API 호출)
- **팔로업**: 최우수 프로젝트를 블로그에 소개, 연장 크레딧 제공, 챔피언 프로그램 초대
- **비용 벤치마크**: 온라인 $5K~$20K; 100명+ 대면 $20K~$100K
