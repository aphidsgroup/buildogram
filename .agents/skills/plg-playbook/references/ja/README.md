# PLG（プロダクトレッドグロース）プレイブック — フリーミアムからエンタープライズへ

プロダクト自身に営業させる。PLG モーションの 3 フェーズで構成。

## フリーミアム設計

- **「アハ体験」は無料で渡す**：無料枠はコア価値を体験できるものに。機能制限だらけのデモにしない
- **課金対象は規模であって価値ではない**：シート数・処理量・管理機能に課金し、ユーザーが愛する機能には課金しない
- **価格心理**：提供前でもエンタープライズ枠を見せてアンカーにする

## セルフサーブ・オンボーディング

- **アクティベーション定義は行動閾値で**（例：実タスクを 2 件以上完了）。「登録済み」は定義にならない
- **Time-to-value を計測**：登録→アクティベーションの全ステップを計測し、寄与しないステップは削除
- **離脱トリガー**：2 回目セッション後 24 時間以内に再訪なし → 自動ウィンバックメール
- **課金アトリビューション**：登録 × アクティベーション × 課金 × 最終ログインの 4 タイムスタンプ

## エンタープライズへの引き渡し（PLG → SLG）

- **シート数ゲート**：カスタム/プライベートデプロイは N シート以上のみ
- **プライベートデプロイの真の理由は 2 つだけ**：データプライバシー + 社内権限システム連携。それ以外は SaaS へ誘導
- **契約規律**：分割払い（3:4:3）、SOW は契約と同時締結、変更は CCB 経由——書面のみが有効
- **リソース配分**：SLG 主導の会社でも約 20% のリソースを PLG に残しパイプラインを育てる

## ベンチマーク

| 指標 | 健全な PLG |
|------|-----------|
| 登録 → 有料 | >5%（Pro-C 5–8%、SMB 10–15%） |
| アクティベーション率 | 40%+（アーリーステージの下限 20%） |
| オンボーディング週次リテンション | >25–30% |
