# Product-Led Growth Playbook — Freemium to Enterprise

Let the product do the selling. Organized by the PLG motion's three phases.

## Freemium Design

What to give away vs. what to gate.

- **Give away the "aha"**: the free tier must deliver the core value moment, not a crippled demo
- **Gate scale, not value**: charge for seats, volume, and admin controls — not for the feature that makes users love you
- **Pricing psychology**: anchor with a visible enterprise tier even before you can serve it
- **Time-boxed trials beat forever-free** when your value compounds with usage data

## Self-Serve Onboarding

- **Activation milestone**: define it as a behavior threshold (e.g., completes ≥2 real tasks), never "signed up"
- **Time-to-value**: instrument every step from signup to activation; kill steps that don't move it
- **Churn trigger**: no return visit within 24h of the second session → automated win-back email
- **Payment attribution**: track four timestamps — signup × activation × payment × last login

## Enterprise Handoff (PLG → SLG)

When self-serve users become enterprise deals:

- **Seat-count gate**: custom/private deployment only above N seats — smaller accounts stay self-serve
- **The two real reasons for private deployment**: data privacy + internal permission-system integration; everything else routes back to SaaS
- **Contract discipline**: staged payments (3:4:3), SOW signed with the contract, changes through a CCB — only paperwork counts
- **Push-pull**: asked for 50 features and can do 40? Commit to 10, stretch to 15 — turn concessions into favors
- **Resource split**: SLG-led companies keep ~20% of resources on PLG as the pipeline feeder

## Benchmarks

| Metric | Healthy PLG |
|--------|-------------|
| Signup → paid | >5% (Pro-C 5–8%, SMB 10–15%) |
| Activation rate | 40%+ (early-stage floor 20%) |
| Onboarding week-retention | >25–30% |
| Free → enterprise pipeline | seat-gate qualified only |
