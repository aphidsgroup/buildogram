# Maker Comment 6 种模板 · 速查手册

> 提炼自 30+ 次 Product Hunt 日冠实战。
> First comment 是发布日转化率最高的一段文字：hunter 00:01 PST 发布后，**00:15 你必须跟上**。

## 硬规则（先记死）

- **<500 字**，T-1 就写好，不要发布日现写。
- **first comment 里不放外链**——触发审核降权。
- 发布后**每 30 分钟回一批评论**，绝不索要 upvote。
- 回复**所有**评论，包括差评——差评认真回复反而是可信度资产。
- 备好 **10 条 FAQ 回复模板**（定价、路线图、对比竞品、数据隐私等高频问题），发布日直接粘贴改写。

## 6 种原型（选 1 个，不要缝合）

### 1. Founder story（创始人故事）
结构：为什么做这个 → 你自己的痛 → 你的解法。
适用：面向大众的产品首选之一；情绪共鸣最强。

### 2. Before / After（前后对比）
结构：没有这个产品时生活/工作什么样 → 用了之后什么样。
适用：价值一眼可见的效率类产品。

### 3. Key-stat hook（数据钩子）
结构：一个爆炸性数字开头 → 展开这个数字背后的故事。
适用：有真实亮眼数据的产品；面向大众的另一首选。

### 4. Honest limitation（坦承不足）
结构：产品做到了什么 → 坦承哪里还不完美 → 路线图。
适用：Beta 阶段产品；"承认不完美"读起来最真实。

### 5. Community thanks（社区致谢）
结构：@ 点名早期测试用户 → 他们帮你改了什么 → 邀请更多反馈。
适用：已有活跃早期社区的产品；点名的用户大概率当场回帖，带起评论区。

### 6. Tech-stack pride（技术选型）
结构：架构/选型决策 → 为什么这么选 → 踩过的坑。
适用：**开发者向产品专属**；HN 气质的 PH 用户吃这套。

## 选型速查

| 受众 | 首选原型 |
|:---|:---|
| 开发者 / 技术人 | #6 Tech-stack pride |
| 大众 / 泛用户 | #1 Founder story 或 #3 Key-stat hook |
| Beta / 早期产品 | #4 Honest limitation |
| 已有测试社区 | #5 Community thanks |

## 发布日评论节奏（配合 first comment）

| 时点（PST） | 动作 |
|:---|:---|
| 00:01 | Hunter 发布 |
| 00:15 | 发 maker first comment |
| 全天 | 每 30 分钟回一批评论 |
| 14:00 | 推 second wave（邮件列表 + 社区） |
| 23:00 | 发感谢帖收尾 |

## T+1 → T+7：评论区的复利

- 回完所有评论（含差评）之后，把 Featured 截图 + 数据做成 case study。
- 进 daily #1 → 写 "how we won" 复盘文并设 canonical 指向自己域名。
- 所有 upvoter 邮箱导入用户池。
- 评论区出现大量质疑 → 转入正式用户访谈流程。

## 反踩雷

- ❌ first comment 塞外链
- ❌ 评论里索要 upvote（任何形式）
- ❌ 只回好评不回差评
- ❌ 忘记 mobile 截图——40% 流量来自手机，评论区第一印象也在手机上
