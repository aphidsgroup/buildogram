# PH Launch Framework — 完整版

## 5 大板块

1. **产品侧**（is your product launch-ready）
2. **内容侧**（what you show）
3. **社群侧**（who will support）
4. **运营侧**（what happens on the day）
5. **复盘侧**（what you do with the outcome）

---

## 板块 1：产品侧检查

- [ ] 落地页 < 3s 加载完，mobile-first
- [ ] 定价页清晰（免费 / 付费分层）
- [ ] 至少 3 个核心 use case 在首屏可见
- [ ] FAQ 含支付、取消、数据、API 可用
- [ ] Demo 视频 ≤ 60 秒，有字幕
- [ ] 3-5 张 PH gallery 图：第 1 张必须讲清"这是什么"

---

## 板块 2：内容侧

### Title（PH 标题 60 字内）
公式：`{Product Name} — {核心能力} for {目标用户}`
例：`Cursor — The AI code editor for professional developers`

### Tagline（副标题 60 字内）
一句话讲清产品怎么帮你省时间 / 赚钱 / 变酷。
❌ "Next-gen AI-powered platform for..."
✅ "Ship PRs 10x faster without reviewing AI code line-by-line"

### Description（800 字内）
结构：
1. 钩子（一个痛点 / 数据 / 金句）
2. "Here's what {product} does"
3. 3 个关键特性 + 每个 1 句话
4. 为什么现在做（时机）
5. CTA

### First Comment（你作为 maker）
- 开场感谢 hunter
- Founder story（为什么做）
- 现在的状态（用户数、里程碑）
- 社群邀请（Discord / Telegram）
- 欢迎反馈的具体问题（不要泛泛的"any feedback?"）

---

## 板块 3：社群侧

### 名单
发布前整理 3 类支持者：
- **铁粉**（付费或重度使用过）→ 私信提醒，请 upvote + comment
- **同行**（其他创始人）→ 群里喊，互相支持
- **冷观众**（newsletter 订阅 / 社媒关注）→ 公开发帖

### 沟通话术模板
```
Hey {name}, we're launching {product} on Product Hunt tomorrow at {time}.
Link: {ph-url}（发布后填）
If you find it useful, an upvote & honest comment would mean the world.
No pressure if it's not your thing.
```

❌ 不要说"please upvote" —— PH 会降权
✅ 用"check it out if interested"

## 板块 4：运营侧（发布日节奏）

| 时间（PST） | 动作 |
|---|---|
| 23:55 T-1 | 最后 asset 检查 |
| 00:01 | Hunter 发布 |
| 00:05 | Maker first comment |
| 00:15 | 给 10 个铁粉发私信 |
| 01:00 | 推特首发 + Reddit 发帖 |
| 06:00 | 第一波 comment 回复 |
| 09:00 | 亚太用户醒来，再推一轮 |
| 12:00 | Dev.to / hacker news 侧推 |
| 14:00 | **美国 peak time**，大 wave |
| 17:00 | 欧洲晚饭时间，回一轮 comment |
| 21:00 | 确认当日排位 |
| 23:00 | 感谢帖 + 数据预告 |

---

## 板块 5：复盘

发布后 48 小时内：
- [ ] 导出 upvoter 邮箱（PH 付费功能）
- [ ] 统计 comment 正负反馈比例
- [ ] 做 1 张数据图：hour-by-hour upvote 曲线
- [ ] 写 case study（至少 5 个学到的）
- [ ] 把亮眼数据填进下一次 pitch

---

## 成功判定

| 档次 | 判据 |
|---|---|
| S（日冠） | Daily #1，≥ 500 upvote |
| A（前三） | Daily top 3，≥ 300 upvote |
| B（曝光） | Featured，≥ 100 upvote |
| C（失败） | < 50 upvote or unfeatured |

C 档不等于失败 —— 如果获得 3 个付费客户也值。
