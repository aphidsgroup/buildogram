# 비개발자 창업자를 위한 오픈소스 마케팅

개발자가 아니어도 오픈소스 프로젝트를 마케팅할 수 있습니다. 기술적 우수성과 시장 인지도 사이의 갭을 메우는 플레이북입니다.

## README를 랜딩 페이지로 활용

README는 당신의 홈페이지입니다. 대부분의 방문자는 10초 안에 스타를 누를지 떠날지 결정합니다.

### 전환되는 구조
| 섹션 | 목적 | 최대 길이 |
|------|------|----------|
| 히어로 배너 | 비주얼 아이덴티티 | 이미지 1장 |
| 원라이너 | 무엇을 하는지 + 누구를 위한지 | 1문장 |
| 데모 GIF | 보여주기, 설명하지 않기 | 15초 |
| 빠른 시작 | 복사-붙여넣기로 체험 | 3단계 |
| 기능 테이블 | 스캔하기 쉬운 혜택 | 5-8항목 |
| 소셜 프루프 | 스타, 사용자, 언론 | 1섹션 |
| 컨트리뷰팅 가이드 | 참여 유도 | 링크 |

### 배지 전략
- 빌드 상태 (신뢰 시그널)
- 버전 번호 (활성 프로젝트)
- Discord 멤버 수 (커뮤니티 규모)
- 라이선스 유형 (법적 명확성)

## 커뮤니티 퍼스트 성장

### Discord/Slack 전략
- 채널 생성: #general, #help, #showcase, #feedback
- 첫 30일간 매일 포스트 (직접)
- 모든 질문에 4시간 내 답변
- 매주 "오피스 아워" 보이스 채팅 운영
- 모든 컨트리뷰터를 공개적으로 축하

### 콘텐츠 캘린더 (비개발자용)
| 주 | 콘텐츠 유형 | 플랫폼 |
|----|------------|--------|
| 1 | "왜 오픈소스로 했는가" 스토리 | 블로그 + Twitter |
| 2 | 사용자 쇼케이스 / 케이스 스터디 | Discord + Reddit |
| 3 | 대안 도구와 비교 | 블로그 + HN |
| 4 | 로드맵 업데이트 + 커뮤니티 의견 | GitHub + Discord |

## 스타 모멘텀

### GitHub Trending 작동 원리
- 총 개수보다 속도(벨로시티)가 중요
- 임계값: 카테고리 트렌딩에 약 50스타/일
- 지속: 임팩트를 위해 3일 이상 연속 트렌딩 필요
- 타이밍: 월요일 런칭이 주중 모멘텀 확보에 유리

### 트리거 전술 (윤리적)
- 같은 날 3개 이상 채널에서 콘텐츠 연계
- 포스트 타이밍은 US 아침 (EST 9AM)
- 기존 사용자에게 스타 요청 (모르는 사람이 아닌)
- r/programming, r/opensource, HN에 크로스포스트

## 컨트리뷰터 퍼널
1. **인지**: "Good first issue" 라벨, 명확한 CONTRIBUTING.md
2. **첫 PR**: 빠른 리뷰 (24시간 내), 따뜻한 환영 댓글
3. **리텐션**: 반복 컨트리뷰터에게 태그, 메인테이너 채팅 초대
4. **어드보커시**: 릴리스 노트에 소개, 굿즈 제공
