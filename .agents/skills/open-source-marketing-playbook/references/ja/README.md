# 非エンジニア創業者のためのオープンソースマーケティング

開発者でなくてもオープンソースプロジェクトのマーケティングは可能です。技術的な優秀さと市場での認知度のギャップを埋めるプレイブック。

## README をランディングページとして活用

README はあなたのホームページです。ほとんどの訪問者は10秒以内にスターを付けるか離脱するか決めます。

### コンバージョンする構造
| セクション | 目的 | 最大長 |
|-----------|------|--------|
| ヒーローバナー | ビジュアルアイデンティティ | 画像1枚 |
| ワンライナー | 何ができるか + 誰向けか | 1文 |
| デモ GIF | 見せる、語らない | 15秒 |
| クイックスタート | コピペで試せる | 3ステップ |
| 機能テーブル | スキャンしやすいベネフィット | 5-8項目 |
| 社会的証明 | スター数、ユーザー数、プレス | 1セクション |
| コントリビュートガイド | 参加を促す | リンク |

### バッジ戦略
- ビルドステータス（信頼シグナル）
- バージョン番号（アクティブなプロジェクト）
- Discord メンバー数（コミュニティ規模）
- ライセンスタイプ（法的明確性）

## コミュニティファースト成長

### Discord/Slack 戦略
- チャネル作成: #general, #help, #showcase, #feedback
- 最初の30日間は毎日投稿（あなた自身が）
- 全質問に4時間以内で回答
- 毎週「オフィスアワー」ボイスチャットを開催
- 全コントリビューターを公開で称賛

### コンテンツカレンダー（非エンジニア向け）
| 週 | コンテンツタイプ | プラットフォーム |
|----|----------------|----------------|
| 1 | 「なぜOSSにしたか」ストーリー | ブログ + Twitter |
| 2 | ユーザーショーケース / ケーススタディ | Discord + Reddit |
| 3 | 代替ツールとの比較 | ブログ + HN |
| 4 | ロードマップ更新 + コミュニティ意見 | GitHub + Discord |

## スター獲得のモメンタム

### GitHub Trending の仕組み
- 合計数より速度（ベロシティ）が重要
- 閾値: カテゴリトレンドに約50スター/日
- 持続: インパクトを出すには3日以上連続でトレンド入り
- タイミング: 月曜ローンチが平日のモメンタムを得やすい

### トリガー戦術（倫理的）
- 同日に3つ以上のチャネルでコンテンツを連携
- 投稿タイミングは US 朝（EST 9AM）
- 既存ユーザーにスターを依頼（見知らぬ人ではなく）
- r/programming, r/opensource, HN にクロスポスト

## コントリビューターファネル
1. **認知**: "Good first issue" ラベル、明確な CONTRIBUTING.md
2. **初 PR**: 高速レビュー（24時間以内）、温かい歓迎コメント
3. **リテンション**: リピーターにタグ付け、メンテナーチャットに招待
4. **アドボカシー**: リリースノートで紹介、グッズ提供
