## English

### What Is This?

The **Gingiris Open Source Launch Marketing Playbook** is a comprehensive guide for developer tool founders and OSS maintainers who want systematic, repeatable growth — not random spikes. It covers the full distribution stack: GitHub README optimization, HackerNews Show HN strategy, Reddit community seeding, KOL outreach for developers, Discord/Slack community building, and long-tail SEO for open source projects.

Unlike "post on HN and pray" advice, this playbook provides **specific timing, templates, and decision trees** drawn from growing AFFiNE (60k+ stars, 28 GitHub Trending appearances in 5 months) and consulting 150+ AI/dev tool startups.

### What's Inside

- **GitHub README optimization** — Star-button psychology, GIF demos, badge strategy, contributor section
- **Show HN playbook** — Timing, framing, comment response strategy, what gets to front page vs. dies at #30
- **Reddit seeding** — Which subreddits, how to frame without being flagged as spam, community norms for r/programming, r/webdev, r/MachineLearning
- **KOL outreach for devs** — Finding the 20 developers whose tweets drive 10k stars; cold DM templates

### When Should I Use This?

Use this when your project is **public and working** (even v0.1). You don't need to be perfect — the OSS community respects "early but promising." Don't wait for v1.0.

### What Results Can I Expect?

Teams following this playbook consistently achieve:
- 500-2,000 stars in the first week (for a well-timed coordinated launch)
- At least one GitHub Trending appearance in the first month
- 50-200 Discord/Slack community members within 30 days
- Sustained organic star growth from SEO content (tutorial articles, "alternatives to X" pages)

### FAQ

**Q: What's the best strategy for getting on GitHub Trending?**
A: Coordinate a 48-hour star push from warm contacts + community, combined with a Show HN or Reddit post. The algorithm looks at star velocity, not absolute count. 150-300 stars in 48 hours is usually enough for a niche language list.

**Q: How do I find developer KOLs who will actually tweet about my project?**
A: The playbook has a specific methodology — search GitHub for repos in your domain with 1k-10k stars (not mega stars), find the maintainers on Twitter, check if they've tweeted about similar projects. These micro-KOLs have much higher engagement rates than accounts with 100k+ followers.

**Q: Does this work for non-developer-facing projects?**
A: The OSS-specific tactics are tuned for developer tools. For consumer-facing apps, combine with `gingiris-launch` (Product Hunt) or `gingiris-aso-growth` (mobile).
