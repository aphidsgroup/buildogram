# 开源宣发话术模板 · 速查手册

> 提炼自 AFFiNE 0 → 60K star 实战原则。模板给的是**结构**，数字和故事必须换成你自己的真实内容——编造数据是最快被社区反噬的方式。

## Show HN 模板

**发帖时机**：周二美东 9am 或周六。

**标题结构**：
```
Show HN: <项目名> – <一句话说清是什么>（<最硬的一个事实：开源/self-hosted/替代谁>）
```

**正文结构（首条评论）**：
1. 我是谁、为什么做这个（1-2 句，真实的痛点起源）
2. 它做什么、和现有方案的区别（技术人视角，不用营销词）
3. 技术选型 1-2 个亮点（HN 用户吃这套）
4. 坦承目前的局限
5. 具体想听什么反馈

**执行规则**：
- 发帖后**守一整天**，回复每一条评论——把 Show HN 当一锤子买卖是反模式。
- 上首页后加一条 SEO 监控：`<项目名> github` 关键词排名。

## Reddit 模板

**选 sub**：3 个——r/selfhosted、r/programming、加一个你领域的细分 sub。

**帖子结构**：
1. 标题：说问题或说成果，不说"我发布了"（社区导向，不是广告导向）
2. 正文：遇到的问题 → 为什么现有工具不满足 → 做了什么 → 开源地址放文末
3. 首条自评：补充技术细节 + 主动问反馈

**执行规则**：
- 每个 sub 先读版规，有的要求 flair 或禁自推日。
- 发完守评论区，认真回每条——Reddit 帖子→star 是直接转化（AFFiNE 单渠道累计 2000+ star）。
- ❌ 只发一次就跑；❌ 同一天轰炸所有 sub。

## KOL 接触话术

**选人标准**：活跃度 > follower 数；3-5 个即可起量。

**私信结构**：
1. 一句话表明看过对方内容（具体到某条，不是套话）
2. 一句话说清项目是什么 + 为什么和 TA 的受众相关
3. 给"一把枪"：附上 PR 文章链接（对方可直接引用，不需要自己想怎么介绍）
4. 明确、低门槛的 ask：看一眼 / 试用 / 有兴趣再聊

**参考基准**：Twitter DM 回复率 3-5%，LinkedIn 10-15%——按这个回复率倒推触达量（持续期每天 3-4 个、月 60-80 个）。

## 竞品 follower 私信模板

**目标人群**：竞品（Notion / Miro 等）Twitter follower 中**公开吐槽过竞品**的用户——已有切换意愿。

**结构**：
1. 引用对方吐槽的具体痛点（证明不是群发）
2. 一句话：我们开源做了 X，正好解决这个
3. Repo 链接 + 欢迎提 issue

**基准**：几百条私信 → 几百个 star 转化。
**红线**：遵守平台频率限制与反 spam 政策；对方不回或明确拒绝就停，绝不追发。

## 通用红线

- ❌ 任何模板里索要 star / upvote
- ❌ 编造用户数、star 数、性能数字
- ❌ 用营销腔写技术社区帖（"revolutionary" / "game-changing" 直接扣分）
- ✅ 所有外发内容里保证 GitHub 回链存在（backlinks 循环）
