# 开源分发渠道矩阵 · 速查手册

> 提炼自 AFFiNE 0 → 60K star 与 Client Project A 0 → 10K star（8 个月）实战。
> 分两段用：Launch 爆发期（T-0 → T+14）与持续增长期（T+14 → T+180，月增 300+ star 节奏）。

## Launch 期渠道（T-0 → T+14）

- [ ] **Product Hunt**——走独立的 launch 时序（见 gr-ph-launch）
- [ ] **Hacker News Show HN**——周二美东 9am 或周六发
- [ ] **Reddit**——选 **3** 个相关 sub：r/selfhosted、r/programming、加一个细分技术 sub
- [ ] **dev.to / Zenn / CSDN**——技术深度文
- [ ] **3–5 个 KOL** 转发（活跃度 > follower 数）

## 渠道优先级矩阵

| 渠道 | 适用阶段 | 效率指标 | 优先级 |
|:---|:---|:---|:---|
| Reddit | 全周期 | 帖子→Star 直接转化（AFFiNE 累计 2000+ star） | ⭐⭐⭐ |
| Hacker News | 爆发期 | 单帖可带 200-500 star | ⭐⭐⭐ |
| Twitter/X | 全周期 | DM 回复率 3-5% | ⭐⭐⭐ |
| LinkedIn | B2B 获客 | DM 回复率 10-15% | ⭐⭐ |
| YouTube | 内容沉淀 | 小额推流（约 30 SGD）即有效 | ⭐⭐ |
| Dev.to / Hashnode | SEO 长尾 | 文章可跨平台分发 | ⭐⭐ |

## 持续增长期节奏（300 star/月 cadence）

| 频率 | 渠道 | 内容类型 |
|:---|:---|:---|
| 每周 2-3 篇 | Dev.to / Medium / 官方博客 | 技术教程、用例解析、对比文章 |
| 每周 1 个 | YouTube | 视频教程 / Demo |
| 每天 1-2 条 | Twitter/X | 技术短内容、GIF、Thread |
| 每周 1 篇 | LinkedIn | 长文 / 案例分享 |
| 每周 2-3 帖 | Reddit | r/selfhosted、r/programming、r/MachineLearning |
| 每天 | Discord / Telegram | 日常互动、答疑 |

配套月度指标：内容 8-12 篇/月 · KOL 触达 60-80/月（每天 3-4 个）· 线上活动 1 次/月 · Contributor PR 10+/月。

## 活动驱动增长

| 类型 | 频率 | 要点 |
|:---|:---|:---|
| 线上 Webinar / AMA | 每月 1 次 | 技术分享 + Live Demo + Q&A（31-34 人规模即有效） |
| 线下 Meetup | 每季度 1 次 | 特定市场深耕（如日本），70 人规模 |
| PR Challenge | 每季度 1 次 | Hacktoberfest / Good First Issue Sprint |
| Hackathon 赞助 | 按需 | 200-500 star/场，印度学生黑客松效率最高 |

## 竞品 follower 触达

- 给竞品（Notion / Miro 等）的 Twitter follower 发私信介绍你的开源替代。
- 实战基准：几百条私信 → 几百个 star 转化。
- 优先找**吐槽竞品**的用户——他们已经有切换意愿。
- 遵守平台频率限制与反 spam 政策，尊重对方的拒绝。

## 渠道顺序与地区分布红线

- **英文 > 多语言 > 纯中文**；第一周海外、第二周国内。
- 同一国家/地区 star 占比 >20% 会失去 Trending 资格——渠道排期时就要控分布。

## 数据漏斗（渠道效果自检）

```
曝光 → 访问 Repo（CTR 2-5%）→ Star（10-30%）→ Clone/Fork（Star 的 5-10%）
  → Contributor（Fork 的 1-5%）→ 付费用户 / Enterprise Lead
```

## 反模式

- ❌ Reddit 只发一次就跑——社群反感，易被 ban
- ❌ 把 Show HN 当一锤子买卖——要回一整天评论
- ❌ 全渠道同一天引爆——分布和节奏都会失控
