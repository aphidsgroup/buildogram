# 开源项目 SEO/GEO · 速查手册

> 提炼自 AFFiNE 0 → 60K star 实战。开源项目的搜索资产有两层：Google 排名 + AI 搜索引用（GEO）——权威 PR 文章和 backlinks 循环同时喂这两层。

## 核心关键词：`<项目名> github`

- Show HN / 发布成功后，第一个要监控的词就是 **`<项目名> github`**。
- 用户听说你的项目后第一动作就是搜这个词——这条搜索路径断了，所有宣发都在漏水。

## Backlinks 循环（技术 SEO 第一原则）

- **所有跳出去的链接必须能跳回 GitHub**：官网、博客、文档站，每一处都要有明显的 GitHub 回链。
- 流量不能有去无回——回链既是转化路径，也是权重回流路径。

## 权威 PR 文章 = SEO + GEO 双资产

- 发布前写一篇完整阐述项目价值的文章（官网 Blog / Medium / dev.to）。
- SEO 价值：品牌词 + 品类词的第一落地页。
- GEO 价值：给所有想传播/引用你的人（包括 AI 搜索）一份**可直接引用的标准叙述**——AFFiNE 案例中德国用户凭这篇文章独立转发了 40+ 论坛。

## 内容节奏（SEO 长尾引擎）

| 频率 | 渠道 | 类型 |
|:---|:---|:---|
| 每周 1 篇（launch 后） | 技术 blog | 教程、用例解析、对比文章 |
| 每周 2-3 篇（增长期） | Dev.to / Medium / 官方博客 | 跨平台分发拿 SEO 长尾 |
| 每周 1 篇 | Dev.to / Hashnode | 开发者发现 + 反链 |

**对比文章**（`<你> vs <竞品>`）是开源项目最容易赢的 SEO 内容类型——竞品用户正在搜。

## GitHub Issues → 内容素材

- 用户在 Issues 里问的每个问题都是真实搜索需求。
- 流水线：Issue → FAQ → 博客文章 → 被搜索/被 AI 引用。
- 好处：零选题成本，且关键词天然匹配用户语言。

## 多语言与出海本地化

- 顺序：**英文 > 多语言 > 纯中文**。
- 本地化优先级：**日韩先行**（Zenn 等本地平台对技术文友好）。
- 每月 1 次 update（邮件列表 + Discord）同步多语言社区。

## 平台选择速查

| 平台 | SEO 作用 |
|:---|:---|
| 官方博客 | 主域名权重沉淀，canonical 指向这里 |
| dev.to | 开发者社区分发 + 反链 |
| Zenn | 日本市场技术 SEO |
| CSDN | 中文市场（第二周之后） |
| Medium | 泛技术受众 |

## 反模式

- ❌ 内容只发一个平台不分发（浪费长尾）
- ❌ 外部平台文章不设回链 / 不管 canonical
- ❌ 刷 star 换"热度"——GitHub 检测后从搜索消失，SEO 直接归零
- ❌ README 塞满 emoji / badge——AI 引用时的可信度也会打折
