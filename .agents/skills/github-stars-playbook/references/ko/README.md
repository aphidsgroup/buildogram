## Show HN 전략

### 효과적인 제목 공식
- "Show HN: [프로덕트] – [8단어 이내 혜택]"
- "Show HN: [X]를 만들었습니다. [공감되는 불만]이라서"
- "Show HN: [X] — [유명 프로덕트]의 오픈소스 대안"

### 타이밍
- 베스트: 화~목요일, EST 10AM
- 피하기: 주말, 월요일 아침, 금요일 오후
- 1시간에 5포인트 미만이면 1회만 재게시

## Reddit 플레이북

### OSS에 최적인 서브레딧
| 서브레딧 | 오디언스 | 포맷 |
|---------|---------|------|
| r/programming | 일반 개발자 | 기술 글 |
| r/opensource | OSS 매니아 | 프로젝트 소개 |
| r/selfhosted | 프라이버시 중시 | "만들었습니다" |
| r/[언어명] | 언어 특화 | 튜토리얼/데모 |
| r/sideproject | 인디 메이커 | 빌드 스토리 |

### 밴 방지 규칙
- 직접적인 "스타 눌러주세요" 금지
- 가치를 먼저, 링크는 끝에
- 1시간 내 여러 서브에 포스트하지 않기
- 댓글에 진정성 있게 참여

## 스타 벨로시티의 과학
- GitHub Trending (일간): 약 50-100스타/일
- GitHub Trending (주간): 3일간 약 200스타
- 알고리즘은 총 개수보다 최근 벨로시티를 중시
- 6시간에 100스타 > 7일에 100스타

## 스프린트 이후 복리 성장
- 주간 콘텐츠 리듬 설정 (최소 주 1회)
- 컨트리뷰터용 "good first issue" 파이프라인 만들기
- 월간 커뮤니티 이벤트 운영 (해커톤, AMA)
- 주간 스타 벨로시티 추적 및 채널 최적화
