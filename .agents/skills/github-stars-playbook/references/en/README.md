## Show HN Strategy

### Title Formulas That Work
- "Show HN: [Product] – [One benefit in ≤8 words]"
- "Show HN: I built [X] because [relatable frustration]"
- "Show HN: [X] — open-source alternative to [known product]"

### Timing
- Best: Tuesday–Thursday, 10 AM EST
- Avoid: Weekends, Monday mornings, Friday afternoons
- Resubmit once if first attempt gets <5 points in 1 hour

## Reddit Playbook

### Best Subreddits for OSS
| Subreddit | Audience | Format |
|-----------|----------|--------|
| r/programming | General devs | Technical post |
| r/opensource | OSS enthusiasts | Project showcase |
| r/selfhosted | Privacy-focused | "I made this" |
| r/[language] | Language-specific | Tutorial/demo |
| r/sideproject | Indie makers | Build story |

### Anti-Ban Rules
- No direct "please star my repo" asks
- Lead with value, link at the end
- Don't post to multiple subs within 1 hour
- Engage in comments genuinely

## Star Velocity Science
- GitHub Trending (daily): ~50-100 stars/day
- GitHub Trending (weekly): ~200 stars over 3 days
- The algorithm weighs recent velocity over total count
- A burst of 100 stars in 6 hours > 100 stars in 7 days

## Post-Sprint Compounding
- Set up weekly content cadence (1 post/week minimum)
- Create "good first issue" pipeline for contributors
- Run monthly community events (hackathons, AMAs)
- Track star velocity weekly and optimize channels
