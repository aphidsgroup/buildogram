# GitHub スターを急速に獲得 — 14日間グローススプリント

オーガニック発見を待つのはやめましょう。これは GitHub Trending 入りを狙う14日間の集中施策です。AFFiNE を0から60k+スターに成長させ、50以上の OSS プロジェクトをコンサルした経験に基づきます。

## 14日間プラン

### Phase 1: 基盤構築（Days 1-3）
| 日 | アクション | チャネル | 目標 |
|----|-----------|---------|------|
| 1 | README 改善（ヒーロー、GIF、クイックスタート） | GitHub | プロフェッショナルな第一印象 |
| 2 | Show HN 記事 + Twitter スレッド作成 | 下書き | 公開準備完了 |
| 3 | サブレディット5つ特定 + 投稿下書き | 下書き | ターゲットコンテンツ |

### Phase 2: 点火（Days 4-7）
| 日 | アクション | チャネル | 目標 |
|----|-----------|---------|------|
| 4 | Show HN 投稿（EST 火曜 10AM） | Hacker News | 50+ポイント |
| 5 | Twitter/X ローンチスレッド（10ツイート） | Twitter | 100+リツイート |
| 6 | Reddit 投稿（時差をつけて3サブ） | Reddit | 合計 200+アップボート |
| 7 | Dev.to + Hashnode クロスポスト | ブログ | SEO + バックリンク |

### Phase 3: 増幅（Days 8-11）
| 日 | アクション | チャネル | 目標 |
|----|-----------|---------|------|
| 8 | テックニュースレター5件にアウトリーチ | メール | 2+掲載 |
| 9 | Discord/Slack コミュニティ3つに投稿 | コミュニティ | 50+クリック |
| 10 | 比較記事作成（vs 代替ツール） | ブログ + Reddit | 高エンゲージメント |
| 11 | Twitter Spaces またはポッドキャスト出演 | 音声 | オーディエンス露出 |

### Phase 4: 維持（Days 12-14）
| 日 | アクション | チャネル | 目標 |
|----|-----------|---------|------|
| 12 | 指標を公開共有（「Day 12 アップデート」） | Twitter | 社会的証明 |
| 13 | コントリビューターに感謝 + まとめ | GitHub + Discord | リテンション |
| 14 | 次スプリント計画 / 自動化構築 | 内部 | 複利的成長 |

## Show HN 戦略

### 効果的なタイトルフォーミュラ
- "Show HN: [プロダクト] – [8語以内のベネフィット]"
- "Show HN: [X]を作りました。[共感できる不満]だったので"
- "Show HN: [X] — [有名プロダクト]のオープンソース代替"

### タイミング
- ベスト: 火〜木曜、EST 10AM
- 避ける: 週末、月曜朝、金曜午後
- 1時間で5ポイント未満なら1回だけ再投稿

## Reddit プレイブック

### OSS に最適なサブレディット
| サブレディット | オーディエンス | フォーマット |
|--------------|--------------|------------|
| r/programming | 一般開発者 | 技術記事 |
| r/opensource | OSS 愛好者 | プロジェクト紹介 |
| r/selfhosted | プライバシー重視 | 「作りました」 |
| r/[言語名] | 言語特化 | チュートリアル/デモ |
| r/sideproject | インディーメーカー | ビルドストーリー |

### BAN 回避ルール
- 直接的な「スターください」は NG
- 価値をリードに、リンクは最後に
- 1時間以内に複数サブに投稿しない
- コメントには誠実にエンゲージ

## スターベロシティの科学
- GitHub Trending（デイリー）: 約50-100スター/日
- GitHub Trending（ウィークリー）: 3日で約200スター
- アルゴリズムは合計数より直近のベロシティを重視
- 6時間で100スター > 7日で100スター

## スプリント後の複利成長
- 週次コンテンツリズムを設定（最低週1本）
- コントリビューター向け "good first issue" パイプラインを作成
- 月次コミュニティイベント開催（ハッカソン、AMA）
- 週次でスターベロシティを追跡しチャネル最適化
