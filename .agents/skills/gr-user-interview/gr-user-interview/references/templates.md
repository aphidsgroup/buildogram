# 用户访谈模板 · 速查手册

> 提炼自 HeyGen 937 场访谈方法论及多个 AI Agent 项目实战。
> 按 Discovery / Validation / Retention 三个阶段组织，直接抄结构、改产品名即用。

## Token Top10 访谈法（日常运营循环）

把访谈从"调研动作"变成"日常运营循环"的最短路径：

1. 后台按 token 消耗 / 用量拉 **Top10 用户 list**（每周刷新）
2. 约**共享屏幕访谈，必须录屏**——看真实操作 > 听口述
3. **Transcript 喂 AI**，自动提取 feature request / bug 直接进 issue 工具
4. **次日回访**：告知"你提的 X 已修/已排期"——反馈闭环本身就是留存杠杆
5. 节奏：**每天至少 2 场**；访谈后截 5-10 分钟关键片段直接给产研

配套：新访谈员先 mock interview + 全程录屏复盘再上真实用户；KOL 访谈二合一（合作 YouTuber 顺便做访谈 + 可用性测试）；一场付费用户访谈可同时产出功能需求 + SEO 素材 + reseller 线索。

## 邀约模板与 1/3/7/14 触达节奏

**四次触达**：Day 1 首发 → Day 3 → Day 7 → Day 14 各跟进一次。
**双通道**：邮件模板 + 邮箱反查 LinkedIn 私信；邮件无回复转 LinkedIn 补一轮。

邀约话术四要素：
1. 说明目的（用宽泛理由如"团队早期想做用户访谈"，**不暴露真实筛选原因**，避免用户带预设表演）
2. 时长承诺（30 分钟）
3. 录屏说明
4. 价值交换（早期权限 / 路线图参与等）

**筛选优先级**：P0 付费用户、高频活跃用户 → P1 竞品用户、流失用户 → P2 注册未付费用户。

## 阶段一：Discovery（探索期问题模板）

**背景了解**
- 你的职业/角色？从哪个渠道知道我们的？用了多久？主要用哪个功能？
- 同类工具还在用哪些？

**工作流探索**
- 这个产品帮你解决什么问题？之前你是怎么做的？现在流程有什么变化？

**痛点挖掘**
- 用的过程中最不爽的地方？遇到过什么 Bug？哪个功能最难用？
- "魔法棒"问题：如果给你一根魔法棒，你最想让产品变成什么样？

**关键技巧**：让用户共享屏幕演示真实操作，观察卡点——屏幕观察 > 用户口述。

## 阶段二：Validation（验证期问题模板）

**竞品对比**
- 用过哪些竞品？各自什么场景下用？我们相比的优劣势？

**付费意愿**
- 之前为类似工具付过费吗？愿意为这个付多少？什么会促使你升级？

**Beta 五问法（结构化收尾）**
1. 你觉得这是一个怎样的产品？（让用户自己贴标签）
2. 最满意的功能是什么？
3. 哪个环节最卡顿或困惑？
4. 如果明天正式上线，你愿意付费吗？为什么？
5. 最希望下个版本加入什么？

**Beta 任务设计**：任务模拟完整工作流而非孤立功能点；任务长度要触发"等待"场景（暴露性能问题）；提供 prompt example；明确说"Beta 不稳定是正常的"。连线测试用 Think Aloud Protocol，利用加载等待期收集用户背景。

## 阶段三：Retention（留存/流失期问题模板）

**流失用户访谈三问**
1. 什么时候开始不用的？当时在做什么？
2. 遇到了什么问题，还是找到了替代品？
3. 如果我们解决了 X 问题，你会回来吗？

**流失归因四类**：性能类（卡顿/超时/崩溃）、工作流断层类（缺下游集成）、ICP 不匹配类、Messaging 不匹配类（触达话术 ≠ 产品实际价值）。

**流失可运营化**：把流失定义成行为（如两次对话后 24h 不回访）→ 自动触发挽回邮件。

## 访谈复盘模板（24 小时内完成）

- 用户画像：角色 / 渠道来源 / 使用时长 / 分级（S/A/B/C）
- 核心工作流：解决什么问题、之前怎么做
- Top 痛点（按提及强度排序）
- Feature request → 进 tracker（标注用户等级 + 付费状态）
- 付费信号：付过什么、愿付多少、升级触发点
- 下一步：回访承诺 / 大使计划邀请 / 转介绍请求

**每 5-10 场汇总一次**：共同需求/痛点 → 更新功能优先级 → 修正用户画像 → 调整产品方向。
