# DevRelプレイブック — デベロッパーリレーションズ: コミュニティ、ドキュメント & イベントSOP

コミュニティ構築からドキュメント戦略、イベント企画までのDevRel完全プレイブック。

## コミュニティ設計

Discord/Slackのセットアップ、モデレーション、エンゲージメントループ：

- **チャネル構造**: #announcements（読み取り専用）、#introductions、#help、#show-and-tell、#feedback、トピック別チャネル
- **モデレーションプレイブック**: スパム自動検出、有害行為のエスカレーションパス、新メンバー向けウェルカムボット
- **エンゲージメントループ**: 週次コミュニティコール、月次ファウンダーAMA、コントリビュータースポットライト
- **ゲーミフィケーション**: ロール昇進（新参者→ヘルパー→コントリビューター→チャンピオン）、アクティビティベースのバッジ
- **ヘルスメトリクス**: DAU/MAU比（目標 >20%）、アクティブメンバーあたりメッセージ数、初回レスポンスまでの時間

## ドキュメント戦略

APIドキュメント、チュートリアル、クイックスタートでコンバージョン：

- **クイックスタートの原則**: デベロッパーが5分以内に動く「Hello World」を得られること
- **チュートリアル構造**: ゴール → 前提条件 → ステップ（番号付き、コピペ可） → ネクストステップ
- **APIリファレンス基準**: 各エンドポイント: 説明、パラメータ、リクエスト/レスポンス例、エラーコード
- **検索最適化**: 「[アクション] + [プロダクト名] + [言語]」クエリでドキュメントが上位表示
- **鮮度の維持**: ドキュメントが実際のAPI動作と乖離したら検知する自動テスト

## イベントプレイブック

ミートアップ、カンファレンス、ハッカソン — ROI追跡：

- **カンファレンス選択**: 総参加者数ではなく、ICP出席密度で優先順位。イベントごとにリードを追跡。
- **登壇戦略**: チュートリアル型トーク（ライブコーディング）はキーノートの3倍コンバートする
- **ブース最適化**: インタラクティブデモ > スライド。QR → サインアップでリード獲得。名刺ではなく。
- **ミートアップ開催**: 月次ローカルミートアップは年次カンファレンスより深い関係を構築
- **ROI公式**: （新規クオリファイドリード + 獲得コミュニティメンバー + 制作コンテンツ）÷ 総コスト

## デベロッパーアドボカシー

コンテンツ制作、ソーシャルプレゼンス、ソートリーダーシップ：

- **コンテンツカレンダー**: ブログ週2本、動画週1本、日次ソーシャルエンゲージメント
- **プラットフォーム優先度**: Twitter/Xはリアルタイム、YouTubeはチュートリアル、BlogはSEO、Dev.toはリーチ
- **オーセンティックなボイス**: デベロッパーとしてデベロッパー向けに書く。マーケ用語禁止。コードを見せる。
- **オープンソースエンゲージメント**: 自社だけでなくエコシステムプロジェクトにもコントリビュート
- **登壇パイプライン**: CFP申請は6ヶ月前から、ローカル→リージョナル→インターナショナルと段階的に

## メトリクス & レポーティング

DORA、コミュニティヘルス、収益への帰属：

- **先行指標**: ドキュメントPV、コミュニティ成長、GitHub Star、ソーシャルメンション
- **遅行指標**: DevRelコンテンツ帰属サインアップ、影響を受けたエンタープライズ案件、NPS
- **DORAメトリクス**: デプロイ頻度、リードタイム、変更失敗率、復旧時間
- **コミュニティヘルススコア**: アクティブメンバー、レスポンスタイム、センチメント、コントリビューターリテンション
- **経営層レポート**: バニティメトリクスではなくビジネスインパクトのナラティブ付き月次サマリー
