# DevRel Playbook — Developer Relations: Community, Docs & Events SOP

Complete Developer Relations playbook — from community building to documentation strategy to event planning.

## Community Architecture

Discord/Slack setup, moderation, and engagement loops:

- **Channel structure**: #announcements (read-only), #introductions, #help, #show-and-tell, #feedback, topic-specific channels
- **Moderation playbook**: Auto-mod for spam, escalation path for toxicity, welcome bot for new members
- **Engagement loops**: Weekly community calls, monthly AMAs with founders, contributor spotlights
- **Gamification**: Role progression (newcomer → helper → contributor → champion), activity-based badges
- **Health metrics**: DAU/MAU ratio (target >20%), messages per active member, time-to-first-response

## Documentation Strategy

API docs, tutorials, and quickstarts that convert:

- **Quickstart principle**: Developer should have working "hello world" in under 5 minutes
- **Tutorial structure**: Goal → Prerequisites → Steps (numbered, copy-paste) → Next Steps
- **API reference standards**: Every endpoint: description, parameters, request/response example, error codes
- **Search optimization**: Docs should rank for "[action] + [your product] + [language]" queries
- **Freshness maintenance**: Automated tests that break when docs drift from actual API behavior

## Event Playbook

Meetups, conferences, hackathons — ROI tracking:

- **Conference selection**: Prioritize by ICP attendance density, not total attendees. Track leads per event.
- **Talk strategy**: Tutorial-style talks (live coding) convert 3x better than keynotes
- **Booth optimization**: Interactive demos > slides. Capture leads via QR → signup, not business cards.
- **Meetup hosting**: Monthly local meetups build deeper relationships than annual conferences
- **ROI formula**: (New qualified leads + community members gained + content produced) ÷ total cost

## Developer Advocacy

Content creation, social presence, and thought leadership:

- **Content calendar**: 2 blog posts/week, 1 video/week, daily social engagement
- **Platform priority**: Twitter/X for real-time, YouTube for tutorials, Blog for SEO, Dev.to for reach
- **Authentic voice**: Write as a developer FOR developers. No marketing speak. Show code.
- **Open source engagement**: Contribute to ecosystem projects, not just your own
- **Speaking pipeline**: CFP submissions 6 months ahead, local → regional → international progression

## Metrics & Reporting

DORA, community health, and attribution to revenue:

- **Leading indicators**: Docs views, community growth, GitHub stars, social mentions
- **Lagging indicators**: Signups attributed to DevRel content, enterprise deals influenced, NPS
- **DORA metrics**: Deployment frequency, lead time, change failure rate, time to restore
- **Community health score**: Active members, response time, sentiment, contributor retention
- **Executive reporting**: Monthly summary with business impact narrative, not just vanity metrics
