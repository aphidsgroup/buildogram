# DevRel 플레이북 — 개발자 관계: 커뮤니티, 문서화 & 이벤트 SOP

커뮤니티 구축부터 문서화 전략, 이벤트 기획까지의 DevRel 완전 플레이북입니다.

## 커뮤니티 설계

Discord/Slack 설정, 모더레이션, 인게이지먼트 루프:

- **채널 구조**: #announcements (읽기 전용), #introductions, #help, #show-and-tell, #feedback, 주제별 채널
- **모더레이션 플레이북**: 스팸 자동 탐지, 유해 행위 에스컬레이션 경로, 신규 멤버 웰컴 봇
- **인게이지먼트 루프**: 주간 커뮤니티 콜, 월간 파운더 AMA, 컨트리뷰터 스포트라이트
- **게이미피케이션**: 역할 승진 (뉴커머→헬퍼→컨트리뷰터→챔피언), 활동 기반 뱃지
- **헬스 지표**: DAU/MAU 비율 (목표 >20%), 활성 멤버당 메시지 수, 첫 응답까지 시간

## 문서화 전략

API 문서, 튜토리얼, 퀵스타트로 전환:

- **퀵스타트 원칙**: 개발자가 5분 이내에 작동하는 "Hello World"를 얻을 수 있어야 합니다
- **튜토리얼 구조**: 목표 → 전제 조건 → 단계 (번호 매기기, 복사-붙여넣기) → 다음 단계
- **API 레퍼런스 기준**: 각 엔드포인트: 설명, 파라미터, 요청/응답 예시, 에러 코드
- **검색 최적화**: "[액션] + [제품명] + [언어]" 쿼리에서 문서가 상위 노출
- **최신성 유지**: 문서가 실제 API 동작과 달라지면 감지하는 자동화 테스트

## 이벤트 플레이북

밋업, 컨퍼런스, 해커톤 — ROI 추적:

- **컨퍼런스 선택**: 총 참석자가 아닌 ICP 참석 밀도로 우선순위. 이벤트별 리드 추적.
- **발표 전략**: 튜토리얼 스타일 토크(라이브 코딩)가 키노트보다 3배 전환율 높음
- **부스 최적화**: 인터랙티브 데모 > 슬라이드. QR → 가입으로 리드 캡처. 명함 아님.
- **밋업 호스팅**: 월간 로컬 밋업이 연간 컨퍼런스보다 깊은 관계를 구축
- **ROI 공식**: (신규 적격 리드 + 획득 커뮤니티 멤버 + 제작 콘텐츠) ÷ 총 비용

## 개발자 애드보커시

콘텐츠 제작, 소셜 프레즌스, 소트 리더십:

- **콘텐츠 캘린더**: 블로그 주 2편, 영상 주 1편, 일일 소셜 인게이지먼트
- **플랫폼 우선순위**: Twitter/X는 실시간, YouTube는 튜토리얼, Blog는 SEO, Dev.to는 도달
- **진정성 있는 보이스**: 개발자로서 개발자를 위해 작성. 마케팅 용어 금지. 코드를 보여주세요.
- **오픈소스 참여**: 자사 프로젝트뿐 아니라 에코시스템 프로젝트에도 기여
- **발표 파이프라인**: CFP 제출은 6개월 전부터, 로컬→리전→글로벌 순으로 진행

## 지표 & 리포팅

DORA, 커뮤니티 헬스, 매출 기여도:

- **선행 지표**: 문서 PV, 커뮤니티 성장, GitHub Star, 소셜 멘션
- **후행 지표**: DevRel 콘텐츠 기여 가입, 영향받은 엔터프라이즈 딜, NPS
- **DORA 지표**: 배포 빈도, 리드 타임, 변경 실패율, 복구 시간
- **커뮤니티 헬스 스코어**: 활성 멤버, 응답 시간, 센티먼트, 컨트리뷰터 리텐션
- **경영진 보고**: 바니티 메트릭이 아닌 비즈니스 임팩트 내러티브가 포함된 월간 요약
