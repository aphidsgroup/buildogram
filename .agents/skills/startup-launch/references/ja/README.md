# スタートアップ ローンチ当日 プレイブック — 時間単位チェックリスト

ローンチ当日は一発勝負です。このプレイブックは、何も漏れがないように時間単位の実行計画を提供します。T-12時間の準備から72時間のポストローンチスプリントまで。

## タイムライン

### T-12h: 最終チェック
- [ ] 全サーバーの負荷テスト完了・安定稼働
- [ ] 決済処理の検証（テストトランザクション）
- [ ] 全リンクの動作確認（LP、ドキュメント、ソーシャル）
- [ ] コピーを2名以上で校正済み
- [ ] アナリティクス/トラッキングの発火確認
- [ ] チーム役割の割当 + バックアップ連絡先

### Hour 0: チャネル同時アクティベーション
| チャネル | アクション | 担当 |
|---------|-----------|------|
| Product Hunt | PST 12:01 AM に公開 | 創業者 |
| Twitter/X | ローンチスレッド（10ツイート以上） | マーケティング |
| メールリスト | ローンチ告知 | グロース |
| Discord/Slack | コミュニティ通知 | コミュニティ |
| Hacker News | Show HN 投稿 | CTO |

### Hours 1-6: リアルタイムモニタリング
- PH ランキングを30分ごとに確認
- 全コメントに15分以内で返信
- サーバー負荷スパイクを監視
- サインアップ速度をトラッキング
- Twitter の会話にエンゲージ

### Hours 6-24: セカンドウェーブ増幅
- KOL/インフルエンサー投稿をずらして有効化
- LinkedIn、Reddit にクロスポスト
- プレス連絡先に「公開しました」を送信
- 初期指標/マイルストーンを公開共有

### 72時間ポストローンチスプリント
- Day 2: ユーザー証言と初期の成果を共有
- Day 3:「ローンチの裏側」コンテンツを公開
- アクティベートしていないサインアップ全員にフォローアップ
- コミュニティエンゲージメントを継続

## 危機管理
| シナリオ | 対応 | 制限時間 |
|---------|------|---------|
| サイトダウン | ステータスページに切替、SNS で通知 | 5分以内 |
| HN でネガティブコメント | 思慮深く対話、防御的にならない | 15分以内 |
| 競合からの攻撃 | 事実誤認でない限り無視、誤りは一度だけ訂正 | 必要に応じて |
| 決済トラブル | 即時修正、手動オンボーディングを提案 | 30分以内 |

Gingiris 制作 — 合計10万以上の初日訪問者を記録した30以上のローンチを統括。
