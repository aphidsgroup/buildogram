# 스타트업 런칭 당일 플레이북 — 시간별 체크리스트

런칭 당일은 단 한 번의 기회입니다. 이 플레이북은 빠짐없이 실행할 수 있도록 시간 단위 실행 계획을 제공합니다. T-12시간 준비부터 72시간 포스트 런칭 스프린트까지.

## 타임라인

### T-12h: 최종 점검
- [ ] 모든 서버 부하 테스트 완료 및 안정 확인
- [ ] 결제 처리 검증 (테스트 트랜잭션)
- [ ] 모든 링크 동작 확인 (랜딩 페이지, 문서, 소셜)
- [ ] 카피 2명 이상 교정 완료
- [ ] 애널리틱스/트래킹 발화 확인
- [ ] 팀 역할 배정 + 백업 연락처

### Hour 0: 채널 동시 활성화
| 채널 | 액션 | 담당 |
|------|------|------|
| Product Hunt | PST 12:01 AM에 공개 | 창업자 |
| Twitter/X | 런칭 스레드 (10개 이상 트윗) | 마케팅 |
| 이메일 리스트 | 런칭 공지 | 그로스 |
| Discord/Slack | 커뮤니티 알림 | 커뮤니티 |
| Hacker News | Show HN 포스트 | CTO |

### Hours 1-6: 실시간 모니터링
- PH 랭킹 30분마다 확인
- 모든 댓글에 15분 이내 응답
- 서버 부하 스파이크 감시
- 가입 속도 추적
- Twitter 대화에 참여

### Hours 6-24: 세컨드 웨이브 증폭
- KOL/인플루언서 포스트를 시차를 두고 활성화
- LinkedIn, Reddit에 크로스포스트
- 언론 연락처에 "출시했습니다" 전송
- 초기 지표/마일스톤을 공개 공유

### 72시간 포스트 런칭 스프린트
- Day 2: 사용자 후기와 초기 성과 공유
- Day 3: "런칭 비하인드" 콘텐츠 발행
- 활성화하지 않은 모든 가입자에게 후속 연락
- 커뮤니티 참여 지속

## 위기 관리
| 시나리오 | 대응 | 제한 시간 |
|---------|------|----------|
| 사이트 다운 | 상태 페이지로 전환, 소셜에서 공지 | 5분 이내 |
| HN 부정적 댓글 | 사려 깊게 대화, 방어적 태도 금지 | 15분 이내 |
| 경쟁사 공격 | 사실 오류가 아니면 무시, 오류는 한 번만 바로잡기 | 필요시 |
| 결제 문제 | 즉시 수정, 수동 온보딩 제안 | 30분 이내 |

Gingiris 제작 — 합산 10만 이상의 첫날 방문자를 기록한 30회 이상의 런칭을 운영.
