# 바이럴 루프 설계 — 리퍼럴·초대·공유 메커니즘

제품은 우연히 바이럴이 되지 않는다. 바이럴은 제품 안에 설계해 넣는 것 — 이 플레이북의 핵심이다.

## 바이럴 루프 해부학

- 세 가지 루프 유형의 작동 원리: 리퍼럴 루프, 초대 루프, 콘텐츠 루프
- 바이럴 루프는 마케팅 캠페인이 아니라 제품 기능이다. 사용자 여정 안쪽에 심는 것

## K-팩터 계산

- 자사의 바이럴 계수를 계산하고, 이를 바탕으로 성장 시나리오를 모델링한다
- K는 '한 코호트의 사용자가 획득 비용 이상의 신규 사용자를 데려오는가'를 알려준다. 루프 기능에 투자하기 전에 반드시 측정할 것

## 인센티브 설계

- 추천하는 사람에게 무엇을 주고, 추천받는 사람에게 무엇을 줄 것인가 — 그리고 '통하지 않는 인센티브'는 무엇인가
- 양면 인센티브는 양쪽 모두가 제품 안에서 실제로 가치를 느끼는 보상이어야 작동한다

## 공유 트리거

- 사용자 여정에서 공유가 '강요'가 아니라 '자연스럽게' 느껴지는 순간을 찾는다
- 잘못된 타이밍의 공유 유도는 마찰이지만, 같은 유도라도 올바른 순간이라면 성장 엔진이 된다

## 케이스 스터디

- Dropbox, Notion, Loom은 어떻게 바이럴 엔진을 만들었나
- 각 엔진 뒤에 있는 메커니즘을 배우고 패턴을 자사에 맞게 적용할 것. 겉으로 보이는 기능만 베끼지 말 것

## B2B vs 2C 보정

이 플레이북의 기본값은 개발자 도구/오픈소스/B2B SaaS. 제품이 2C(컨슈머·교육·앱·게임)라면:

- 핵심 지표를 D1/D7/D30 리텐션, 활성화율, 바이럴 계수 K로 교체 (MRR/CAC/LTV가 아니라)
- 콜드스타트 채널을 버티컬 커뮤니티, 숏폼 영상, 버티컬 KOL로 교체 (Product Hunt / Hacker News / LinkedIn이 아니라)
- 국가별 채널 데이터가 포함된 완전한 2C 적응 가이드는 `gingiris-seo-geo/references/2c-adaptation.md` 참조
