# BUILDOGRAM FULL ECOSYSTEM USER JOURNEYS

# Purpose

This document defines the complete user journeys for Buildogram's full ecosystem.

Buildogram is not a single-user product.

It serves:

- Construction customers
- Property owners
- Tenants
- Buyers
- Builders
- Contractors
- Architects
- Suppliers
- Site engineers
- Sales team
- Operations team
- Finance team
- Content team
- Maintenance vendors

Each journey should connect to:

- Lead capture
- CRM
- Revenue
- Property records
- Partner ecosystem
- Material marketplace
- Property Passport
- Long-term retention

Core principle:

**Every user journey should create trust, data, revenue or ecosystem power.**

# Journey 1: Construction Customer

## User Type

Land owner or property owner who wants to construct a building.

## Entry Points

- Homepage
- Home construction page
- Cost calculator
- BOQ audit page
- Plan review page
- Instagram reel
- WhatsApp referral
- Partner referral
- Google search

## User Intent

- Understand construction cost
- Find trustworthy execution
- Compare quotes
- Avoid cheating
- Get quality construction
- Track project properly

---

## Journey Flow

### Step 1: Discovery

User lands on:

- /build/home-construction
- /ai-tools/construction-cost-estimator
- /boq-audit

User sees:

- Build with proof
- BOQ clarity
- BQS quality checks
- Customer dashboard
- Property Passport

CTA:

- Calculate Cost
- Upload BOQ
- Book Consultation

---

### Step 2: Lead Capture

User submits:

- Name
- Phone
- Location
- Plot area
- Built-up area
- Construction type
- Budget range
- Timeline
- Requirement

System creates:

- Lead
- Lead activity
- Lead source
- Lead score

---

### Step 3: Sales Qualification

Sales executive calls user.

Qualification checks:

- Land ownership
- Budget readiness
- Timeline
- Approval status
- Existing plan/BOQ
- Decision maker
- Expected service model

Lead status becomes:

- Contacted
- Qualified
- Consultation scheduled

---

### Step 4: Consultation

Buildogram explains:

- Execution model
- BOQ model
- Quality proof
- Material support
- Customer portal
- Property Passport
- Rental/resale value

Outcome:

- Site visit scheduled
- BOQ audit requested
- Plan review requested
- Proposal requested

---

### Step 5: Site Visit / Document Collection

Collect:

- Site photos
- Location
- Measurements
- Soil/context notes
- Existing drawings
- Approval documents
- Customer expectations

System stores:

- Site visit record
- Documents
- Photos
- Notes

---

### Step 6: Proposal / BOQ

Buildogram creates:

- BOQ version
- Cost summary
- Timeline
- Scope
- Payment milestones
- Quality promise
- Material plan

Customer sees:

- BOQ in portal
- PDF proposal
- Explanation call

---

### Step 7: Project Onboarding

Once accepted:

System creates:

- Customer account
- Project
- Property record
- Property Passport draft
- Milestones
- BOQ approved version
- Document vault

Customer receives:

- Login
- Project dashboard
- Welcome message

---

### Step 8: Execution Tracking

During construction:

- Engineer uploads daily report
- Photos/videos stored
- Quality checks submitted
- Materials verified
- Issues tracked
- Payments linked to milestones

Customer sees:

- Progress
- Quality
- BOQ
- Documents
- Materials
- Issues

---

### Step 9: Handover

Buildogram provides:

- Final quality report
- Final BOQ summary
- Warranty details
- Completion media
- Documents
- Property Passport

---

### Step 10: Post-Handover

Buildogram offers:

- Maintenance
- AMC
- Rental listing
- Resale readiness
- Interior upgrade
- 360 tour
- Property Passport sharing

## Revenue Points

# Final User Journey Principle

Every user journey should pull people deeper into Buildogram's ecosystem.

A construction lead can become a material buyer.
A material buyer can become a construction customer.
A partner can become a content collaborator.
A rental owner can become a maintenance customer.
A resale seller can become a Property Passport customer.
A completed construction customer can become a rental/resale/maintenance customer.

This is Buildogram's ecosystem advantage.
