# Buildogram Team Onboarding Checklist

This checklist is for onboarding new operations staff, sales managers, and site engineers onto the Buildogram platform.

## 1. System Access & Security
- [ ] Receive Buildogram Ops email account (`name@buildogram.in`).
- [ ] Set up 2-Factor Authentication (2FA) for the email account.
- [ ] Receive Ops Portal login credentials.
- [ ] Sign the internal Non-Disclosure Agreement (NDA) regarding client property data and contractor pricing configurations.

## 2. Platform Familiarization
- [ ] Log into `/ops` and verify dashboard access.
- [ ] Review the `BUILDOGRAM_INTERNAL_TEAM_USAGE_GUIDE.md` playbook.
- [ ] Complete a dummy run:
  - [ ] Create a dummy lead in `/ops/leads`.
  - [ ] Convert the dummy lead into an active project.
  - [ ] Log a dummy daily progress update.
  - [ ] Review the generated client-side view to understand what the user sees.
- [ ] Read through the `/ops/help` page SOPs.

## 3. Tool Integration
- [ ] Ensure WhatsApp Web is connected to the designated Ops support number.
- [ ] Verify access to the internal Slack/Discord channels for engineering escalations.
- [ ] Verify read-only access to the Razorpay dashboard (if role permits) to cross-check failed transactions.

## 4. Policy Review
- [ ] Understand the 2-Hour SLA for responding to new leads.
- [ ] Review the strict BQS (Buildogram Quality Standard) policy: No escrow release without verified structural tests.
- [ ] Review the Data Privacy protocol: Never share a client's Property Passport data or floor plans with external third parties without written consent.
